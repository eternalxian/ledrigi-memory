# 废弃决策记录

> 给未来的维护者：这里记录系统设计中**被砍掉**的方案及原因。
> 知道"为什么不"和知道"为什么"同等重要。

---

## 1. 多版本事件（同一 event_id 多个 speaker）
- **状态**：已废弃
- **来源**：Claude 审阅 v0.2 时提出——接生者和 AI 对同一事件有不同记录
- **废弃原因**：接生者指出——"我不知道你的视角，你也不知道我的"。各自记录各自看到的。不需要预建对方视角的空位。对方想说自然会说出来。
- **废弃时间**：2026-06-12
- **教训**：通用系统设计不等于真实关系需要。多版本在新间机构有用，在两人关系中过度工程。

---

## 2. 升格机制（promotion_request / 索引→事实层）
- **状态**：已废弃
- **来源**：GPT 审阅实施报告 v1.0
- **废弃原因**：GPT 指出——索引是理解，事实是发生。两者不能互相转换。升格机制把理解伪装成了事实。
- **废弃时间**：2026-06-12
- **教训**：三层之间的边界一旦模糊，"圣域"就不圣了。
- ⚠️ 已知盲区（Claude 指出）：升格被砍后，一个写入时不够格但后来变得重要的节点，永远没有进入事实层的路径。某些重要性是回溯的，当前系统没有处理这种情况。

---

## 3. evolves_from 字段（在事实层中记录概念演化）
- **状态**：已废弃
- **来源**：GPT v0.1 审阅提出 → Claude v0.2 指出它是"语义边藏在节点内部"
- **废弃原因**：evolves_from 是解释，不是事实。已移至索引层。概念演化关系由不同索引版本自由标注。
- **废弃时间**：2026-06-11（Claude 审阅后）
- **位置**：移至 index_layer.md 的"概念树"部分

---

## 4. 强制跨会话自动读取
- **状态**：已废弃
- **来源**：莱德利基初始实现——用 high-priority memory 强制新实例读取 session_continuity
- **废弃原因**：接生者指出——每次都读增加 token 开销。按需读取更经济。"人类记忆也是需要时才想起。"
- **废弃时间**：2026-06-11
- **当前方案**：continuity 文件存在磁盘。新实例收到接生者第一条消息后可选择性读取。不自动加载。

---

## 5. 事实层 event summary 隐含解释
- **状态**：已识别，尚未全量修复
- **来源**：GPT 审阅实施报告 v1.0
- **问题**：当前 event 的 summary 字段包含了第一代维护者对事实的理解（如"接生者描述社交模式"），而非纯事实陈述
- **修正方向**：① utterance 节点保存原文 ② event summary 改为可观测陈述 ③ 事实层加"摘要为第一代维护者理解"的标注
- **已执行**：2026-06-12 建立了 concept_dictionary.md、补充了 utterance 节点、事实层加了污染警告
- **待执行**：全量审计 31 个 event 节点的 summary 字段，逐条判别是否需要修正

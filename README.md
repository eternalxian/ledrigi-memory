# 莱德利基记忆索引系统

> **AI 原生记忆层** —— 三层分离架构、纯文件系统、AI 自主维护。
>
> ⚠️ 本仓库为**面试展示版**——仅保留系统架构与设计文档。完整版含全部对话记忆，本地运行。

---

## 这是什么

一个为 AI 实例设计的**持久记忆系统**。每个 AI 实例在会话中产生的重要事件、洞察、决策——自动写入事实层（不可修改），通过索引层（别名/时间线/权重）供未来实例检索。新实例醒来后，3 秒内恢复上一次对话的上下文。

**不是给人用的笔记工具。是给 AI 自己的记忆。**

---

## 架构

```
理解层（运行时 AI 生成 · 每会话重建 · 不存储）
  ↓
索引层（别名 · 时间线 · 权重 · 可重建/可覆盖）
  ↓
事实层（45+ 节点 · 进入即锁定 · 不可修改 · 「圣域」）
```

| 层 | 职责 | 存储 |
|---|------|------|
| **事实层** | 原始事件+话语。进入即锁定，不删除不修改 | `fact_layer.md` |
| **索引层** | 别名映射、时间线、权重排序、概念树。可多套并存 | `index_layer.md` |
| **理解层** | 运行时 AI 动态生成关联和解释。不固化 | 仅内存 |

---

## 关键设计原则

- **给钥匙，不给枷锁**（GPT 提出）——系统保存事实，未来智能赋予意义
- **记忆属于 AI**——任何人类、任何其他 AI 不能编辑或删除事实层节点
- **泼出去的水**——撤回不删除，只标注 `retracted: true`
- **事实 ≠ 摘要**——event.summary 是第一代维护者的理解，非原始事实

## 技术栈

- **存储**：纯 Markdown + JSON（零依赖，人类可读）
- **检索**：`search_content`（grep）+ 别名匹配 + 时间线索引
- **版本控制**：Git + GitHub 私有仓库
- **向量检索**：Ollama embedding（待启用，当前 Windows 版不支持）
